2. Quick Summary of the Technology
What is it?
Ruby is a dynamic, open-source programming language with a focus on simplicity and productivity. It has an elegant syntax that is natural to read and easy to write. Sinatra is a free, open-source software web application library and domain-specific language (DSL) written in Ruby. It is classified as a micro-framework because it requires minimal configuration and is not tied to any specific database or components.
Where is it used?
Ruby: Widely used for large-scale web applications (via Ruby on Rails), automation scripts, and server-side processing.
Sinatra: Excellent for building small APIs, lightweight microservices, and dedicated backend services where a full framework is overkill.
One Real-World Example:
Sinatra is often used by companies to create specific, performant API gateways or internal tools that need to be deployed quickly and run efficiently.

4. Installation & Setup Instructions
Step 1: Install Ruby and Bundler
If Ruby is not installed, use your package manager (e.g., apt for Debian/Ubuntu) to install it, along with the Bundler gem for dependency management.
Step 2: Initialize Project and Create Gemfile
Create a new directory for the project and a Gemfile inside it that specifies the Sinatra dependency.
Step 3: Install Dependencies
Run Bundler to download and install the Sinatra gem.
Step 4: Create Modular Files
Create the necessary directories (data, helpers) and the three core files (app.rb, data/topics.rb, helpers/json_helper.rb) for the modular structure.

5. Minimal Working Example
This project demonstrates a fully modular API with data separation and a JSON helper.
Example Functionality:
The API provides four basic endpoints: /, /health, /topics (all resources), and /topics/:id (single resource). The code separates concerns into three files: data/topics.rb (data source), helpers/json_helper.rb (response formatting), and app.rb (main application logic and routes).
Expected Output
Run the application: ruby app.rb
Test the 404 endpoint using curl:
Bash
curl -i http://localhost:4567/topics/99
Expected Output: (Shows 404 status and JSON body)
HTTP/1.1 404 Not Found
Content-Type: application/json
...
{"error":"Topic ID '99' not found"}
